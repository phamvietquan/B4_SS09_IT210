# Bài 4: Form Đăng ký Nhà bán hàng 3 bước (Wizard Form)

## Phần 1 - Phân tích và đề xuất giải pháp

### Giải pháp 1: Dùng input hidden (Client-side)

Cách hoạt động:
- Sau khi nhập dữ liệu ở Bước 1, dữ liệu được gửi về server
- Server trả lại form Bước 2 kèm theo dữ liệu cũ dưới dạng:
  <input type="hidden" name="fieldName" value="...">
- Dữ liệu tiếp tục được gửi qua lại giữa các bước thông qua form submit

Ưu điểm:
- Không tốn bộ nhớ server
- Dễ triển khai, không cần cấu hình phức tạp

Nhược điểm:
- Dữ liệu nằm trên phía client
- Người dùng có thể dùng F12 để sửa giá trị trước khi submit

---

### Giải pháp 2: Dùng @SessionAttributes (Server-side)

Cách hoạt động:
- Dữ liệu từ Bước 1 được lưu vào Session
- Các bước tiếp theo lấy lại dữ liệu từ Session
- Đến Bước 3, gom toàn bộ dữ liệu và lưu vào Database

Ưu điểm:
- Dữ liệu được giữ ở server
- Người dùng không thể chỉnh sửa trực tiếp bằng F12

Nhược điểm:
- Tốn bộ nhớ server
- Nếu nhiều user bỏ dở giữa chừng sẽ gây tồn đọng dữ liệu (risk tràn RAM)

---

## Phần 2 - So sánh và lựa chọn

| Tiêu chí                     | input hidden (Client)         | @SessionAttributes (Server)     |
|----------------------------|------------------------------|---------------------------------|
| Bảo mật                    | Thấp (có thể sửa bằng F12)   | Cao (không sửa trực tiếp)       |
| Độ phức tạp khi code       | Thấp                         | Trung bình                      |
| Rủi ro phình bộ nhớ server | Không                        | Có (nếu nhiều user bỏ giữa chừng) |

---

## Kết luận

- Nếu ưu tiên đơn giản và tiết kiệm tài nguyên: dùng input hidden
- Nếu ưu tiên bảo mật dữ liệu: dùng @SessionAttributes

Giải pháp an toàn hơn trong thực tế:
- Sử dụng @SessionAttributes kết hợp với cơ chế dọn dẹp session (timeout hoặc clear khi user hủy)
- Đồng thời validate dữ liệu ở server trước khi lưu DB
